사무 자동화 웹 서비스 작업 기록

프로젝트 위치
- E:\CODEX\office_automation

현재 구성 파일
- index.html
- styles.css
- script.js

구현된 전체 방향
- 로그인 없이 바로 사용하는 정적 웹 형태로 시작
- 상단 메뉴바에 드롭다운 메뉴 구성
- 각 메뉴 클릭 시 해당 작업 패널로 화면 전환

상단 메뉴 구조
1. 엑셀 파일 변환
   - 파일 병합
   - 파일 분할
   - 그래프 제작
2. 인사 정보 관련
   - 입/퇴사 분석
   - 재직 인원 분석
3. 근태 관련
   - 연차관리대장 제작
4. 급여 관련
   - 급여 비교분석
   - 사업계획
   - 퇴직금 계산
5. 게시판

파일 병합 기능 구현 내용
- 다수의 엑셀 파일 업로드 가능
- 컬럼 행 번호 입력 가능
  - 예: 첫 번째 행이 컬럼이면 1
- 병합 방식 2가지 구현

1. 하나의 시트에 병합
- 업로드 파일들의 첫 번째 시트를 기준으로 처리
- 컬럼명이 서로 다를 수 있으므로 새 컬럼은 오른쪽 끝에 추가
- 없는 값은 빈 셀로 유지
- 컬럼 행 아래 데이터 행을 순차적으로 이어 붙임
- 결과 파일명: merged_single_sheet.xlsx

2. 시트 구분 병합
- 업로드 파일별로 하나의 시트 생성
- 시트명은 파일명을 기반으로 생성
- 결과 파일명: merged_by_sheets.xlsx

파일 분할 기능 구현 내용
- 다수의 엑셀 파일 업로드 가능
- 컬럼 행 번호 입력 가능
- 분할 방식 3가지 구현

1. 행 기준 분할
- 사용자가 입력한 행 수 단위로 분할
- 예: 1000 입력 시 1000행씩 분할
- 각 업로드 파일 기준으로 여러 개의 엑셀 파일 생성
- ZIP으로 다운로드
- 결과 파일명: split_by_rows.zip

2. 컬럼 기준 분할
- 기준 컬럼명을 입력
- 해당 컬럼 값이 같은 행끼리 하나의 파일로 묶음
- 값이 다르면 다른 파일로 분할
- 각 업로드 파일 기준으로 처리
- ZIP으로 다운로드
- 결과 파일명: split_by_column.zip

3. 시트 기준 분할
- 업로드 파일의 각 시트를 각각 별도 엑셀 파일로 분리
- ZIP으로 다운로드
- 결과 파일명: split_by_sheet.zip

엑셀 처리 라이브러리
- SheetJS CDN 사용
  - https://cdn.jsdelivr.net/npm/xlsx@0.18.5/dist/xlsx.full.min.js
- JSZip CDN 사용
  - https://cdn.jsdelivr.net/npm/jszip@3.10.1/dist/jszip.min.js

서식 관련 결정 사항
- 전체 서식(색상, 폰트, 테두리, 병합 셀 등)은 유지하지 않음
- 표시형식 중심으로만 일부 유지
- 특히 날짜 셀은 원본 셀의 표시형식이 있으면 유지하도록 구현
- 숫자에 대해 천단위 콤마를 일괄 적용하는 기능은 제거함
- 현재 정책:
  - 원본 셀에 표시형식이 있으면 그것을 유지 시도
  - 일반 정수/소수는 별도 통일 서식 적용하지 않음

현재 한계
- 브라우저 기반 처리이므로 Excel의 모든 서식을 완전히 유지하지는 못함
- 병합/분할은 주로 값과 표시형식 중심
- 파일 병합 및 파일 분할의 행/컬럼 기준 처리는 기본적으로 첫 번째 시트를 기준으로 동작
- 시트 기준 분할은 전체 시트를 각각 별도 파일로 분리
